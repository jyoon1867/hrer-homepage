# 가비아 hrer.kr → Vercel + Resend 연결 가이드
작성: 2026-04-19

**순서 중요**: Vercel 도메인 먼저 → Resend 도메인 → 홈페이지 URL 일괄 변경 → 배포

---

## Step 1. Vercel 도메인 연결 (10분)

### 1-1. Vercel에서 도메인 추가
1. https://vercel.com 로그인 → `hrer-homepage` 프로젝트
2. 상단 `Settings` 탭 → 좌측 `Domains`
3. 입력창에 **`hrer.kr`** 입력 → `Add` 클릭
4. 다시 `www.hrer.kr` 입력 → `Add` 클릭 (선택, www 붙여도 열리게)

### 1-2. Vercel이 보여주는 DNS 레코드 확인
Vercel이 2가지 방식 중 하나를 요구할 것:

**방식 A: A 레코드** (가장 일반적)
```
Type: A
Name: @ (또는 공란)
Value: 76.76.21.21
```

**방식 B: Nameserver 통째로** (권장 안 함 — 가비아 고정 네임서버 쓰시는 게 편함)

**www 서브도메인**
```
Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

### 1-3. 가비아 DNS 설정

1. https://dns.gabia.com 접속 → 로그인
2. `DNS 설정` → hrer.kr 선택 → `설정` 클릭
3. 기존 레코드 중 `@` 의 A 레코드가 있으면 **삭제하거나 Vercel IP로 수정**
4. 다음 레코드 추가:

| 타입 | 호스트 | 값/위치 | TTL |
|------|--------|---------|-----|
| A    | @      | `76.76.21.21` | 기본 (3600) |
| CNAME | www   | `cname.vercel-dns.com.` (끝에 점 포함) | 기본 |

5. `확인` → `저장`

### 1-4. DNS 전파 대기 (5분~24시간, 보통 10~30분)
- Vercel 대시보드에서 자동으로 감지
- 녹색 체크 표시 뜨면 완료
- SSL 인증서는 Vercel이 자동 발급 (Let's Encrypt)

### 1-5. 접속 확인
- https://hrer.kr 열어보기 → 홈페이지 정상 노출되면 OK
- https://www.hrer.kr → hrer.kr 로 자동 리다이렉트 되는지 확인

## Step 2. Resend 도메인 인증 (15분)

### 2-1. Resend 계정·도메인 추가
1. https://resend.com 로그인 (없으면 가입 — 무료 플랜으로 충분)
2. 좌측 메뉴 `Domains` → `Add Domain` 클릭
3. 도메인 입력: **`hrer.kr`**
4. Region 선택: `ap-northeast-1` (Tokyo) 추천
5. `Add` 클릭

### 2-2. Resend가 보여주는 DNS 레코드 3~4개
보통 이런 형태로 나옵니다:

| 타입 | 호스트 (Name) | 값 (Value) |
|------|---------------|-----------|
| MX   | send.hrer.kr (또는 send) | `feedback-smtp.ap-northeast-1.amazonses.com` (우선순위 10) |
| TXT  | send.hrer.kr (또는 send) | `v=spf1 include:amazonses.com ~all` |
| TXT  | resend._domainkey.hrer.kr | `p=MIGfMA0GCSq...` (긴 DKIM 키) |
| CNAME | _dmarc.hrer.kr (선택) | `v=DMARC1; p=none;` |

**주의**: 실제 값은 Resend 화면에서 복사해야 합니다. 위는 예시 형식입니다.

### 2-3. 가비아 DNS에 레코드 추가

Vercel 설정한 같은 페이지(`dns.gabia.com`)에서:

| 타입 | 호스트 | 값 | TTL |
|------|--------|-----|-----|
| MX   | send  | Resend가 준 값 (우선순위 10) | 기본 |
| TXT  | send  | `v=spf1 include:amazonses.com ~all` | 기본 |
| TXT  | resend._domainkey | Resend가 준 긴 DKIM 키 | 기본 |

**중요**
- 가비아에서 호스트 입력 시 `.hrer.kr` 생략 (자동으로 붙음). `send`만 입력.
- MX 레코드는 우선순위 10 지정
- TXT 값의 따옴표 유무: 가비아 기본적으로 자동 처리. 값만 그대로 복사

### 2-4. Resend에서 인증 확인
- 가비아 저장 후 `Resend 대시보드 → Verify` 버튼 클릭
- 10~30분 내 `Verified ✓` 표시
- 안 되면 1시간 더 대기 후 재시도

### 2-5. API 키 발급
1. Resend 대시보드 → `API Keys` → `Create API Key`
2. 이름: `HRer Production`, Permission: `Sending access`, Domain: `hrer.kr`
3. 생성된 키 복사 (한 번만 보임)

### 2-6. Vercel에 환경변수 등록
1. Vercel → `hrer-homepage` → Settings → Environment Variables
2. 이미 `RESEND_API_KEY`가 있으면 **Edit → 새 키로 교체**
3. 없으면 Add:
   - Key: `RESEND_API_KEY`
   - Value: (2-5에서 복사한 키)
   - Environment: Production + Preview + Development 모두 체크
4. Save → Deployments에서 Redeploy

도메인 인증 + 환경변수 교체 후:

**테스트 1: 의뢰 접수 (견적형)**

2. 테스트 의뢰 제출
3. contact@hrer.kr 메일함 확인 — 관리자 알림 도착
4. 입력한 이메일에도 접수 확인 도착

**테스트 2: 결제 (노무자문)**
1. https://hrer.kr/order 접속
2. Toss 테스트 카드로 결제 (`4242-4242-4242-4242`, 모두 임의)
3. 완료 화면 확인
4. contact@hrer.kr · 본인 이메일 양쪽 도착 확인

## Step 5. Toss Payments 상용 키 발급

### 5-1. 법인 사업자 등록 완료 후
Toss Payments는 사업자 등록번호·계좌가 필요합니다.

### 5-2. 상용 키 발급
1. https://www.tosspayments.com 가입·로그인
2. 사업자 정보·계좌 등록
3. 심사 완료 후 `개발자센터 → 상점 관리 → API 키` 로 이동
4. `라이브 클라이언트 키`와 `라이브 시크릿 키` 복사

### 5-3. Vercel 환경변수 교체
| Key | Value | 환경 |
|-----|-------|------|
| `TOSS_SECRET_KEY` | `live_sk_xxxxx...` (시크릿 키, 서버용) | Production+Preview+Dev |

### 5-4. order.html 클라이언트 키 교체

`order.html` 395행 근처의 테스트 키를 운영 키로 변경 필요:
```javascript
const clientKey = 'live_ck_xxxxx...';  // Toss에서 복사
```

이것도 지민이 라이브 키 발급 후 저에게 알려주시면 바로 반영합니다.

## 💡 트러블슈팅

**도메인 연결 안 됨**
- 가비아 DNS 전파 10~30분 대기
- `dig hrer.kr` 또는 https://dnschecker.org 에서 A 레코드 전파 확인
- Vercel에서 수동 재검사 (Refresh 버튼)

**Resend Verify 안 됨**
- DKIM TXT 값이 여러 줄일 수 있음 → 모두 따옴표 안에 한 줄로
- `send` 호스트와 `resend._domainkey` 구분 실수 많음
- 가비아 TXT 레코드 중복 확인

**이메일 받는 사람이 스팸함으로 감**
- DMARC 레코드 추가 (Resend에서 안내)
- SPF·DKIM 모두 pass 여부 Gmail 원본 보기로 확인

**Toss 결제 안 됨**
- 테스트 키(test_ck_...) vs 상용 키(live_ck_...) 혼용 주의
- 클라이언트 키는 `order.html`, 시크릿 키는 Vercel env 따로
