## 1. 완성된 기술 시스템

### 1-1. 홈페이지 (hrer.kr, 현재 vercel.app)
- 9개 페이지 + 관리 대시보드
- 5종 서비스 플로우 (결제 1 + 견적 4)
- 챗봇 (5중 방어막, Gemini 3단계 폴백, SSE 스트리밍)
- 의뢰→결제→이메일→Supabase 전체 루프
- SEO 기초 (canonical·sitemap·robots·noindex)
- 실명 노출 전면 제거 (동료 발견 리스크 차단)

### 1-2. Supabase DB (v1~v5 스키마 적용 완료)
| 버전 | 내용 | 상태 |
|------|------|------|
| v1 | chat_sessions, chat_messages, chat_feedback, chat_misses, orders | ✅ |
| v2 | orders 답변 저장 컬럼 (response_body 등) | ✅ |
| v3 | taxonomy_id, utm, referrer, raw_input, 결제 상세 | ✅ |
| v4 | ai_citations, ai_model, learning_examples 테이블 | ✅ |
| v5 | page_views, chatbot_events, 집계 뷰 | 👉 오늘 적용 |

### 1-3. 자문봇 Phase 1 (Claude API 없이 가능한 준비)
`HRer_AI/advisor_bot/` 7개 모듈 전부 완성:
- `prompt_builder.py` — CLAUDE.md + rules/ tier별 통합
- `rag.py` — HRer_DB 5단 검색
- `fewshot.py` — 자문패턴 Few-shot 추출
- `assembler.py` — 전체 입력 조립
- `parser.py` — LLM 출력 → body + citations 분리
- `diff.py` — A vs B 격차 측정 (embedding 없이)
- `dry_run.py` — Claude 호출 없이 파이프 시뮬레이션

**법인 완료 후 `generator.py`만 작성하면 즉시 가동 가능.**
실측 비용: 의뢰 1건 $0.045~0.053.

### 1-4. 유입 추적 레이어 (오늘 신규)
- `api/track.js` — Edge, 프라이버시 최소 수집 (IP 해시만)
- `analytics.js` — 16개 HTML 전부 자동 삽입
- `bot-admin.html` — "📈 유입 분석" 카드 추가
  - 소스별 방문자 (네이버·구글·티스토리·LinkedIn 자동 라벨링)
  - 페이지별 조회
  - 채널별 퍼널 (방문→챗봇→의뢰)

### 1-5. 법적 문서
- `privacy.html` 5-1조 신설: AI·챗봇·학습 데이터 활용 고지
- 위탁사 테이블에 Supabase·Gemini·Claude·Resend 추가
- 해외 이전 고지 + 동의 철회권 명시

## 2. 마케팅 전략 (2026-04-20 최종 확정 ★)

### 2-1. 전략 전환 요약

**폐기된 가정** (v1, 2026-04-19):
- 티스토리 우선 (구글 SEO 중심)
- LinkedIn·인스타 병행으로 B2B 리치
- 주 4~8편 포스팅

**현재 유효한 전략** (v2, 2026-04-20):
- **네이버 블로그 최우선** (한국 HR 담당자 검색 패턴)
- **일 1편 × 90일 = 90편** (매일 포스팅)

2. **"자문 필요한 순간에만 검색한다"**
   - 검증: 챗봇 접속 시간대 분포 (월요일 아침·밤에 몰리는지)
3. **"첫 5명이 나머지 100명을 데려온다"**
   - 검증: 첫 10명 고객 중 재의뢰·추천 발생 수
